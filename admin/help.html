What is ETL?

ETL, extract, transform, & load , is the process used to migrate data from one database to another. The data was read from one database, converted into a form that can be combined with data from other databases, and written into the target database. The process relied on the use of a lookup table or data transformation rules to create a uniform template upon which all the data can model. This allows a user to access multiple databases at the same time and this exactly what Colliers needs. For this project we worked with four separate databases and, using the ETL process, created a single data warehouse. The result was hosted on a secure server at UW. The dashboard interacts with the data warehousing producing real-time search results.

What is an ontology?

An ontology is a description of concepts and relationships that exist formally for a user or a community of users (Tom Gruber, 2009). Ontologies are generally looked upon a taxonomies that contain hierarchies of classes, definitions, and relationships. However, they are to be used for knowledge sharing and reuse. Google uses an ontology to make connection to words that users have in their search queries. They use the semantic understanding of the search query to help optimize the results and it returns. This technology can be leveraged with ETL to help connect seemingly unconnected information.
